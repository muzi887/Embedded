# 五位密码传递三组数字方案（A:1~8, B:1~300, C:1~40）

## 1) 可行性说明

- 组A：1~8（8种）
- 组B：1~300（300种）
- 组C：1~40（40种）
- 时间信息：每10分钟为一个单位，`T = H×6 + ⌊分/10⌋`，`T ∈ [0, 143]`（每日144档）

信息组合数：`8 × 300 × 40 × 144 = 13,824,000`  
编码方式：`N = N_base + T×27`，其中 `N_base` 为信息序号，`T` 为10分钟档位  
`N_base` 最大 95999，`T×27` 最大 3861，故 `N` 最大 99860  
5位十进制密码容量：`00000~99999` 共 `100,000` 种  
因此可实现一一映射、无混淆、可逆解析，且**密码与时间关联**。

## 2) 双方约定

- 输入范围：
  - `A ∈ [1,8]`
  - `B ∈ [1,300]`
  - `C ∈ [1,40]`
- 输出内容：
  - `P`：5位密码（左侧补0）
  - `TS`：生成时间（`YYYY-MM-DD-HH:mm`，需含分钟以确定10分钟档位）

## 3) 编码算法（发送端）

### Step 1：生成信息序号 N_base（0~95999）

`N_base = (A-1)×12000 + (B-1)×40 + (C-1)`

### Step 2：计算10分钟档位 T（0~143）

从 `TS = YYYY-MM-DD-HH:mm` 中取小时 `H`、分钟 `mm`，计算  
`T = H × 6 + ⌊mm / 10⌋`

### Step 3：生成与时间关联的序号 N（0~99860）

`N = N_base + T × 27`

### Step 4：生成5位密码 P

`P = N`，按 5 位字符串格式输出，不足补 `0`。

### Step 5：发送格式

建议：`P | TS`

例如：`30178 | 2026-03-18-14:25`

## 4) 解码算法（接收端）

收到 `P, TS` 后：

### Step 1：恢复 N

`N = int(P)`

### Step 2：计算10分钟档位 T（或尝试相邻档位）

从 `TS` 计算 `T = H×6 + ⌊mm/10⌋`。若设备时间不同步，可尝试 `T-1`、`T`、`T+1`（连续上下10分钟）三个值。

### Step 3：反算信息序号 N_base

`N_base = N - T × 27`

### Step 4：反解 C、B、A

- `C = (N_base mod 40) + 1`
- `q = N_base // 40`
- `B = (q mod 300) + 1`
- `A = (q // 300) + 1`

### Step 5：范围校验

- `A` 必须在 `[1,8]`
- `B` 必须在 `[1,300]`
- `C` 必须在 `[1,40]`
- `N_base` 必须在 `[0, 95999]`

超界即判定该档位假设不成立，尝试下一档位；若三档位均超界则消息异常。

## 5) 示例

### 示例1：普通值（密码与时间关联）

输入：`A=3, B=150, C=25, TS=2026-03-18-14:25`

`N_base = (3-1)×12000 + (150-1)×40 + (25-1) = 29984`  
`T = 14×6 + ⌊25/10⌋ = 84 + 2 = 86`  
`N = 29984 + 86×27 = 29984 + 2322 = 32306`  
`P = 32306`

发送：`32306 | 2026-03-18-14:25`

解码：`N_base = 32306 - 86×27 = 29984`，可还原 `A=3, B=150, C=25`

### 示例2：同一信息不同时间产生不同密码

输入：`A=1, B=30, C=15`，时间分别为 10:00、10:10、10:20

| 时间 TS | T | N_base | N | 密码 P |
|---------|---|--------|---|--------|
| 2026-03-15-10:00 | 60 | 1174 | 2794 | 02794 |
| 2026-03-15-10:10 | 61 | 1174 | 2821 | 02821 |
| 2026-03-15-10:20 | 62 | 1174 | 2848 | 02848 |

**同一组信息在不同10分钟档位产生不同密码。**

### 示例3：边界值

输入：`A=8, B=300, C=40, TS=2026-03-18-23:50`

`N_base = 95999`，`T = 23×6 + 5 = 143`  
`N = 95999 + 143×27 = 99860`  
`P = 99860`

可正确反解回 `A=8, B=300, C=40`。

### 示例4：同一密码在不同时间假设下的解码（逆向示例）

**问题**：密码 `02794`，若给定的时间信息分别为 2026-03-15-10:20、2026-03-15-10:10、2026-03-15-10:00，解算结果如何？

**解码过程**：`N = 2794`，对每个假设的 T 计算 `N_base = N - T×27` 并反解：

| 给定时间 | T | N_base | 解析结果 |
|----------|---|--------|----------|
| 2026-03-15-10:20 | 62 | 1120 | A=1, B=29, C=1 |
| 2026-03-15-10:10 | 61 | 1147 | A=1, B=29, C=28 |
| 2026-03-15-10:00 | 60 | 1174 | A=1, B=30, C=15 |

**结论**：同一密码在不同时间假设下可得到多组不同的合法解析，且均在有效范围内。

### 示例5：连续上下10分钟与多解歧义

**问题**：连续的三个10分钟档位（上一档、当前档、下一档）对应同一密码时，是否都能结算出一组有用信息？

**说明**：
- 若发送端与接收端时间误差在 ±10 分钟内，尝试 `T-1`、`T`、`T+1`（连续上下10分钟）时，**至少有一个**会对应正确的 T，从而得到正确的那组信息。
- 但如示例4所示，**多个档位可能同时给出合法解析**，接收端无法仅凭密码和时间戳唯一确定哪一组是“真正”的发送内容。
- 因此存在**多解歧义**：能保证“至少算出一组合法结果”，但不能保证“唯一确定是哪一组”。

## 6) 实际实施

由于离线各设备的时间实际不能同步，解算密码时需对 `TS` 计算的档位尝试 `T-1`、`T`、`T+1`（连续上下10分钟）三个值，分别计算 `N_base` 并反解。若多组结果均在有效范围内，则存在歧义，需结合业务逻辑或其它约定进一步判断。
