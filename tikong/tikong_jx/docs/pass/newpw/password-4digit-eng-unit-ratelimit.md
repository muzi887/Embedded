# HH-TOTP-4 落地说明：选单元 + RateLimit（软硬件对照）

更新时间：2026-07-19  
读者：平台/App 软件工程师、门禁固件与硬件工程师  
基线方案：[password-4digit-design.md](./password-4digit-design.md)  
通俗流程：[password-4digit-flow-simple.md](./password-4digit-flow-simple.md)

本文固定一种**可联调的落地剖面**，不讨论其它变体（园门不选单元、明文楼层等）。

---

## 0. 剖面一句话

**园区门必须先选定单元（或扫码得到 `unit_id`），再验 4 位码；所有验票设备必须启用 RateLimit。**  
口令算法、时间槽、密钥层级与设计文档一致；本文补齐双方职责、误开数字、容量结论与联调清单。


| 项          | 本剖面取值                                  |
| ---------- | -------------------------------------- |
| 方案名        | HH-TOTP-4                              |
| 口令         | 4 位十进制 `0000`–`9999`（比对按整数 `0`–`9999`） |
| `SLOT_SEC` | `600`                                  |
| `SKEW`     | `1`（试 `slot-1, slot, slot+1`）          |
| `VER`      | `0x01`                                 |
| 园门         | **强制** `selected_unit_id`              |
| RateLimit  | **强制**（推荐默认见 §4）                       |

## 1. 角色与职责


| 方               | 负责                                                             | 不负责           |
| --------------- | -------------------------------------------------------------- | ------------- |
| **云 / App（软件）** | 持 `K_c`；派生 `K_u`；按 §3 生成码；展示有效期；下发密钥与配置；园门 UI/扫码协议里带单元         | 现场 RTC；继电器/呼梯 |
| **园区门固件**       | UI/扫码取 `selected_unit_id`；用 `K_c` 派生 `K_u`；统一搜索验票；RateLimit；开门 | 呼梯            |

| **电梯固件**        | 持本机 `K_u`；搜索验票；**唯一层**才呼梯；RateLimit                            | —             |
| **硬件 / 生产**     | 安全写入密钥与 `campus_id`/`unit_id`/`F_MAX`；RTC；读头/键盘；门锁与呼梯接口        | 自创拆位规则        |

## 2. 算法输入约定

### 2.1 HMAC 消息：二进制整数，不是十进制字符串

```text
msg (11 字节，无 key_id 时) =
  ver        : 1 字节  u8
  campus_id  : 4 字节  u32 大端
  unit_id    : 2 字节  u16 大端
  floor      : 1 字节  u8     // 1 … F_MAX
  slot       : 4 字节  u32 大端
```

**错误示例（禁止）**：把 `"1,3,5,202607191430"` 或 ASCII 园区号拼进 SHA1。

### 2.2 单元钥派生

```text
K_u[16] = HMAC-SHA256(K_c, "unit" || U16BE(unit_id)) 的前 16 字节
```

- `"unit"`：4 字节 ASCII → `75 6E 69 74`。
- 其后紧跟 `unit_id` 的 2 字节大端。

### 2.3 通行码

```text
code = DynTrunc31(HMAC-SHA1(K_u, msg)) mod 10000
```

- App 展示：`"%04d"` 字符串（可含前导零）。
- 读头/键盘：可收字符串 `"0384"`，固件转为 `uint16_t code` 再比。
- `DynTrunc31`：RFC 4226 §5.3（与 HOTP 相同）：用摘要末字节低 4 bit 选偏移，取 4 字节，清最高 bit，得 31-bit 无符号整数。

### 2.4 时间

```text
slot = floor(unix_utc_seconds / 600)
```

一律 **UTC Unix 秒**；禁止用本地时区墙钟字符串参与 `slot`。

## 3. 生成与校验（本剖面）

### 3.1 平台生成（软件）

```text
Generate(campus_id, unit_id, floor, t_gen, K_c) → "3847"
  K_u = Derive(K_c, unit_id)
  slot = t_gen / 600
  code = Trunc4(HMAC-SHA1(K_u, msg(ver,campus,unit,floor,slot)))
```

建议 UI 提示有效至约 `(slot + 2) * 600` 对应的 UTC 时间（含 ±1 槽容差，约 20～30 分钟）。

### 3.2 公共验票搜索（固件）

在已知 `K_u, campus_id, unit_id` 时：

```text
N_trials = F_MAX × (2×SKEW+1) = F_MAX × 3
对每个 (floor∈[1,F_MAX], δ∈{-1,0,+1}) 计算 Trunc4，与输入 code 比较
```


| 设备        | 成功条件                  | 动作                         |
| --------- | --------------------- | -------------------------- |
| 园区门（已选单元） | ≥1 次命中                | 开门                         |
| 单元门       | ≥1 次命中                | 开门                         |
| 电梯        | **恰好 1 个不同 floor** 命中 | `CallFloor(该层)`；0 或多层 → 拒绝 |


### 3.3 园区门：选单元（本剖面强制）

```text
1. 访客选择或扫码 → selected_unit_id ∈ 本园允许列表
2. K_u = Derive(K_c, selected_unit_id)
3. 再跑 §3.2 搜索
4. 未选单元 → 直接拒绝（禁止静默穷举全园单元）
```

单元门 / 电梯：本机已有 `unit_id`，**不必**再选单元。

## 4. RateLimit（本剖面强制）

### 4.1 推荐默认参数（软硬件可并列配置，但现场必须启用）


| 参数              | 推荐值   | 说明              |
| --------------- | ----- | --------------- |
| `RL_MAX_FAIL`   | `5`   | 连续失败次数上限        |
| `RL_WINDOW_SEC` | `300` | 统计窗口 5 分钟       |
| `RL_LOCK_SEC`   | `300` | 触发后锁定时长（可与窗口相同） |


行为约定：

1. **每次**开始验票前：若处于锁定 → 直接拒绝（可短鸣），**不**增加失败计数（或按产品约定）。
2. 验票失败（含电梯「多层命中」）→ 失败计数 +1；达上限 → 进入锁定。
3. 验票成功 → 失败计数清零，解除锁定。
4. 计数与锁定状态：**每台设备本地独立**（按设备，不按用户）。
5. 断电后：推荐保持锁定截止时间（若有 RTC/非易失）；若无法保持，上电可清零但须在说明书中声明。

平台侧若提供「远程解锁」，属运维增强，非算法必需。

### 4.2 RateLimit 不改变单次误开公式，但压低盲猜

见下一节。

---

## 5. 重码 / 误开率（选单元 + RateLimit）

口令空间：`S = 10000`。

### 5.1 单次随机输入的误接受率（FAR）

选单元后，园门与单元门搜索空间相同：

```text
FAR_once ≈ (F_MAX × 3) / 10000
```


| `F_MAX` | 试验次数 | 约 FAR（单次）        |
| ------- | ---- | ---------------- |
| 20      | 60   | **0.60%**        |
| 30      | 90   | **0.90%**        |
| 40      | 120  | **1.20%**（文档基准例） |
| 60      | 180  | **1.80%**        |


含义：攻击者（或访客）**随便输一个 4 位数**，在未锁定前提下，约有上表概率被当成「本单元某层某邻近槽的合法票」。

电梯额外要求唯一楼层：单次误开略低于上表，量级同阶；若碰巧两层同时撞码则拒绝。

### 5.2 加上 RateLimit 后的「盲猜可行性」

无 RateLimit 时，期望尝试次数约 `10000 / (F_MAX×3)`（`F_MAX=40` 时约 **83 次**）即可能误开一次。

本剖面 `5 次 / 5 分钟` 锁定：


| 指标            | 约值（F_MAX=40）             |
| ------------- | ------------------------ |
| 每窗口最多有效猜测     | 5 次                      |
| 单窗口内至少误开一次的概率 | `1 − (1−0.012)^5 ≈ 5.9%` |
| 持续盲猜          | 多数时间被锁；需很长时间才有可观累积成功概率   |


结论：**选单元把 FAR 压在 ~1% 量级；RateLimit 使键盘穷举在工程上不可行。**  
二者缺一不可：只选单元不限次，仍可快速试；只限次但不选单元，园门若穷举多单元会 FAR 飙升。

### 5.3 「两张合法票 digests 相同」

不同 `(floor, slot, …)` 截断碰撞 ≈ **1/10000**。属截断固有，与是否选单元无关。

### 5.4 合法访客误拒

时钟在 ±1 槽内、密钥与 `F_MAX` 一致时，合法码应通过。误拒主因：RTC 偏差过大、配置不一致、RateLimit 误伤（连续输错）。属工程问题，不是 FAR。

## 6. 支持设备数量（本剖面）

**没有「最多 N 台门机」的算法硬上限。** 4 位码不编码设备台数。


| 维度       | 结论                                                 |
| -------- | -------------------------------------------------- |
| 园区数      | 每园独立 `K_c`；台数不限                                    |
| 单元数      | `unit_id` 为 u16；**选单元后**单元多少不增加单次 FAR              |
| 同单元门/梯台数 | 共用 `K_u`；**台数不增加 FAR**                             |
| 真正受限     | `F_MAX`（越大 FAR 越高）；MCU 上 `F_MAX×3` 次 HMAC-SHA1 的耗时 |


**固件性能经验**：`F_MAX=60` → 最多 180 次 HMAC-SHA1。F103 类 MCU 软件 SHA1 通常可在百毫秒量级内完成（以实现为准，联调时打时间戳）。若超时，可优化实现或对单元门在「已选楼层」时缩小搜索。

**容量表述（给产品/销售）**：

> 在「园门选单元 + RateLimit + F_MAX≤40」剖面下，设备台数按项目扩容即可；安全指标由单次约 1.2% FAR + 限次共同保证，而非由「小区最多几台机」保证。

## 7. 配置与密钥下发清单

### 7.1 每台设备必须一致的字段


| 字段                      | 园区门        | 单元门    | 电梯     |
| ----------------------- | ---------- | ------ | ------ |
| `campus_id`             | ✓          | ✓      | ✓      |
| `unit_id`               | 不固定（用 UI）  | ✓ 本机单元 | ✓ 同单元  |
| `F_MAX`                 | ✓（与单元配置一致） | ✓      | ✓      |
| `K_c`                   | ✓          | ✗      | ✗      |
| `K_u`                   | 运行时派生      | ✓ 预置   | ✓ 同单元门 |
| 允许单元列表                  | ✓          | —      | —      |
| RateLimit 参数            | ✓          | ✓      | ✓      |
| `SLOT_SEC`/`SKEW`/`VER` | 与云编译期或配置一致 | 同左     | 同左     |


### 7.2 禁止

- 全小区一把钥；跨单元共用 `K_u`。
- 园门未选单元却对所有单元循环验票（本剖面禁止）。
- 平台用字符串拼 msg、固件用二进制（或相反）。
- 平台用本地时区算 slot、固件用 UTC（或相反）。

## 8. 联调与验收

### 8.1 固定向量

实现后用同一组已知输入生成黄金向量，写入仓库测试（示例结构，数值由实现填入）：

```text
K_c   = <16 bytes hex>
unit  = 3
K_u   = HMAC-SHA256(...)[0:16] = <预期>
campus= 1, floor=5, slot=<固定>
msg   = <11 bytes hex>
code  = <0000–9999>
```

平台单测与 MCU 单元测试比对 `K_u`、`msg`、`code` 三项全等。

### 8.2 现场验收用例


| #   | 步骤            | 期望                                       |
| --- | ------------- | ---------------------------------------- |
| 1   | 合法票，园门选对单元    | 开门                                       |
| 2   | 合法票，园门选错单元    | 拒绝，计失败                                   |
| 3   | 合法票，单元门 / 电梯  | 开门 / 呼正确层                                |
| 4   | 改 1 位码        | 拒绝                                       |
| 5   | 连续失败 5 次      | 锁定，期间合法票也不开（或按产品是否允许锁定中放行——本剖面默认**不放行**） |
| 6   | 锁定结束再输合法票     | 通过                                       |
| 7   | 设备钟拨慢/快约 8 分钟 | 仍可通过；拨快 40 分钟应失败                         |


### 8.3 上报建议

成功/失败上报带：`device` 身份、`selected_unit_id`（若有）、是否 RateLimit 锁定、电梯命中 `floor`（成功时）。便于软硬件联查。

## 9. 分工检查表

**软件**

- 生成路径与 §2 msg 字节序一致  
- 展示 4 位前导零；有效期文案与槽宽一致  
- 园门产品流程：必须选/扫单元后再提交码  
- 密钥下发按园/单元隔离；禁止万能钥  
- 提供与固件共用的测试向量

**硬件 / 固件**

- 园门未选单元拒识  
- RateLimit 默认启用且参数可配置  
- 单元门/电梯搜索与唯一层逻辑正确  
- RTC→UTC slot；±1 槽  
- `F_MAX` 与云一致；HMAC 耗时可接受  
- 密钥区写保护 / 不通过调试口明文导出（按产品等级）
