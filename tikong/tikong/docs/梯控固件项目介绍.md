## 一、项目定位

本工程是 **电梯门禁 / 梯控访问控制系统** 的主控固件：扫码或刷卡后，在本地完成验签、黑名单与有效期校验，再按设备类型执行院门禁 / 楼门禁 / 梯控权限（含 **RS485** 下发楼层等）；同时经 **4G 模块** 与物联网平台交互（MQTT 物模型：属性、事件、服务）。

| 项 | 说明 |
| --- | --- |
| 项目类型 | 嵌入式梯控 / 门禁访问控制 |
| MCU | STM32F407VET6（Cortex-M4，512KB Flash，192KB RAM） |
| IDE / 编译器 | Keil MDK-ARM / ARM Compiler 5 |
| 调试下载 | J-Link |
| 工程文件 | `USER/Template.uvprojx` |
| 运行模型 | 裸机轮询 + 中断/DMA（无 FreeRTOS） |

## 二、系统架构

```text
┌─────────────────────────────────────────────────────────────┐
│  云平台（MQTT）                                              │
│  属性设置 / 事件上报 / 服务调用（黑名单、透传、公钥等）        │
└───────────────────────────┬─────────────────────────────────┘
                            │ USART3 @ 115200（4G 模块）
┌───────────────────────────▼─────────────────────────────────┐
│  STM32F407 主控                                              │
│  · 读头 JSON → CommContrl（设置 / 权限 / 校时 / 限层等）      │
│  · SHA1 验签 / 黑名单 / RTC 有效期 / 限层                    │
│  · JSON 上报与服务回执；可选 4G AT 配网序列                   │
└─┬───────────┬───────────┬───────────┬───────────────────────┘
  │ UART4     │ UART5     │ USART2    │ USART1
  │ @ 9600    │ @ 9600    │ @ 9600    │ @ 115200
  ▼           ▼           ▼           ▼
读头 1      读头 2     RS485 梯控    PC 调试
```

分层与目录对应关系（权威说明见 [PROJECT_STRUCTURE.md](../PROJECT_STRUCTURE.md)）：

| 层级 | 目录 | 职责 |
| --- | --- | --- |
| 入口 / 板级编排 | `USER/` | `main`、`board_init`、`app_run`、中断、Keil 工程 |
| 通信业务轮询 | `app/comm/` | `QRProcessUart4/5`、`G4GProcess`、`PCProcess`、AT 序列 |
| 领域逻辑 | `app/`、`app/utils/` | `data_handler`、`blackList`、`Live_data_r`、`cmd` |
| 片外器件驱动 | `drivers/` | I2C、EEPROM、DS1302、4G、继电器等 |
| 板级资源 | `board/` | 定时器、HC595、扩展 IO、`RTC` 抽象、`floor_ctrl` |
| 第三方 | `third_party/` | cJSON、SHA1 |
| 外设驱动层 | `SYSTEM/` | 延时、sys、多路 USART（仅 DMA/ISR/缓冲，不含 `*Process`） |
| 标准外设库 | `FWLIB/` | STM32 SPL |
| 核心 | `CORE/` | Cortex-M4 启动与 CMSIS |

**约定**：`SYSTEM/usart` 不依赖 `app`；业务只通过各 UART 头文件中的缓冲与 `RX_Complete` 标志取数。

## 三、硬件接口一览

| 接口 | 用途 | 波特率 | 引脚（要点） |
| --- | --- | --- | --- |
| USART1 | PC 调试 | 115200 | PA9 TX / PA10 RX；DMA + 空闲中断 |
| USART2 | RS485 ↔ 梯控板 | 9600 | PA2 TX / PA3 RX，PA1 收发控制 |
| USART3 | 4G ↔ 云平台 | 115200 | PB10 TX / PB11 RX；接收 JSON / 上报；AT 配网 |
| UART4 | 读头 1（扫码/刷卡） | 9600 | PC10 TX / PC11 RX |
| UART5 | 读头 2（扫码/刷卡） | 9600 | PC12 TX / PD2 RX |
| I2C / 三线 | EEPROM、RTC | — | EEPROM 持久化密钥/黑名单等；RTC 经 `RtcChip_*`（当前实现为 **DS1302**） |
| TIM1～5 | 周期任务 | — | 校时标志、限层检查、蜂鸣等（见 `board/timer.c`） |
| 蜂鸣器 / 继电器 / HC595 | 本地提示与输出 | — | `board_init` 中初始化 |

## 四、核心能力

### 4.1 读头命令（本地主路径）

读头数据经 **UART4 或 UART5** 累积为 `{...}` JSON 帧，解析 `type` / `data` / `uid` 后调用 `CommContrl`（`app/utils/cmd.c`）。CSV 载荷首字符为命令号（与旧版「命令 1～6」编号不完全相同，以源码为准）：

| 命令字 | 功能 | 说明 |
| --- | --- | --- |
| `1` | 设备参数设置 | `Cmd_Setting`；常触发 4G AT 序列，成功后再写 EEPROM 并复位 |
| `2` | 权限访问 | `Cmd_Permission`：验签 → 黑名单 → 时间窗 → 按 `g_device_type` 走 Gate / Unit / Elevator |
| `6` | 设置时间及漂移 | `Cmd_Set_Time` |
| `7` | 常开 / 限层设置 | `Cmd_Set_OpenLimit` → `floor_ctrl` / EEPROM |

权限成功时：扫码（`order_type` 为 `1`）走 `qr_Reply`，刷卡（`0`）走 `card_Reply`；梯控路径可经 RS485 下发楼层，并经 USART3 上报平台。

黑名单增删等能力仍在 `BlackUserListAdd` / `Del` / `Clear` 等函数中，亦可由云端 JSON 服务触发。

### 4.2 云端联动（4G / MQTT）

USART3 收包后：`G4GProcess`（`app/comm/g4g_comm.c`）→ `parseSerialData`（`app/Live_data_r.c`），与读头路径共用业务能力，结果 JSON 回传平台。

- 处理流程说明：[G4GProcess.md](../USER/G4GProcess.md)  
- AT / 主题参考：`app/comm/AT_reference.txt`、`app/comm/uart3_at_sequence.*`  

主要包括：属性设置 / 上报应答、事件上报、服务（RS485 透传、黑名单、公钥、校时等）。

### 4.3 安全与存储

- **SHA1**：权限等业务数据验签（设备公钥，`third_party/sha1`）  
- **黑名单**：内存表 + EEPROM（`app/blackList`、`drivers/eeprom`）  
- **时间窗**：`RtcChip_GetTime` 对照载荷有效期（时间不准则通行失败）  
- **限层**：`board/floor_ctrl` + 主循环定时检查限层截止时间  
- **配置写回**：部分设置命令等 AT 全部成功后再落 EEPROM（`Cmd_Setting_OnAtSequenceDone`）

## 五、主程序流程

初始化：

1. `NVIC_PriorityGroupConfig`  
2. `board_init()`：延时、USART1～5、定时器、DS1302、继电器、4G、EEPROM/`ReadKey`、HC595 等  
3. `app_init()`：`loadDataFromEEPROM`、RTC、限层状态  

主循环（`USER/main.c`）：

```text
while (1) {
    IWDG_Feed();
    if (g_uart3_at_sequence_request)
        uart3_at_sequence_poll();   // 4G AT 配网；成功则写 EEPROM 并复位
    else {
        app_poll();                 // 正常业务
        // TIM 标志：GetNetTime / 限层时间检查 …
    }
}
```

`app_poll()`（`USER/app_run.c`）当前为：

```text
PCProcess();        // USART1
QRProcessUart5();   // 读头 UART5
QRProcessUart4();   // 读头 UART4
G4GProcess();       // USART3 云端
```

### 数据流（摘要）

```text
读头通行：
  UART4/5 → QRProcessUart* → CommContrl(cmd '2')
    → 验签/黑名单/时间窗 → Gate|Unit|Elevator → 本地动作 / RS485 → 上报(USART3)

云端指令：
  4G → USART3 → G4GProcess → parseSerialData → 业务 → JSON 回执

设备配置：
  读头 cmd '1' 或平台 → Cmd_Setting → AT 序列 → EEPROM → 复位

黑名单 / 限层：
  云端服务或本地命令 → EEPROM + 内存 / floor_ctrl
```

流程图：[main_flowchart.mmd](../USER/main_flowchart.mmd)、[main_flow.puml](../USER/main_flow.puml)。

## 六、工程目录

```text
tikong/
├── CORE/           # Cortex-M4 核心与启动
├── FWLIB/          # STM32 标准外设库
├── SYSTEM/         # delay / sys / usart（驱动层，无 *Process）
├── third_party/    # cJSON、sha1
├── drivers/        # 片外器件（eeprom、iic、ds1302、4G…）
├── board/          # 定时器、HC595、RTC 抽象、floor_ctrl…
├── app/            # 领域逻辑 + utils/cmd + comm/*Process
├── USER/           # main、board_init、app_run、Keil 工程
└── docs/           # 介绍、入门规划与部分模块说明
```

编译：Keil 打开 `USER/Template.uvprojx`，目标芯片 STM32F407VETx；Include Path 需含 `third_party`、`drivers`、`board`、`app`、`app/utils`、`app/comm`。生成 HEX/BIN 后经 J-Link 下载。
