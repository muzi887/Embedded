## 一、为什么需要调试接口

写好的程序要 **下载进 Flash**，出错时要 **停在断点、看变量、单步执行**。这些靠 MCU 片内 **调试模块** + 外部 **调试器**（战舰常用 **J-Link** 或 **ST-Link**，接板载 **JTAG/SWD 口**）。

| 方式 | 能做什么 | 不能替代谁 |
|------|----------|------------|
| **SWD / JTAG + 仿真器** | 烧录、断点调试、读内存 | —（入门主力） |
| **USB 转串口**（战舰常板载） | 看日志、`printf` | **不能**当调试器烧录 |
| **串口 ISP（BOOT0=1）** | 用 Bootloader 写 Flash | 无断点调试；要改启动跳线 |

**口诀**：**仿真器 + SWD = 写进去、查得到；USB 串口 = 看打印。**

---

## 二、JTAG 是什么

**JTAG**（Joint Test Action Group）最初是电路板 **边界扫描测试** 标准，后来成为 MCU 常用的 **多线调试/编程接口**。

在 Cortex-M / STM32 上，指经调试口访问内核与存储器的一套协议。

### 2.1 常见信号线

| 信号 | 方向（相对 MCU） | 作用 | F103 默认脚 |
|------|------------------|------|-------------|
| **TCK** | 入 | 测试时钟（JTCK） | **PA14** |
| **TMS** | 入 | 模式选择（JTMS） | **PA13** |
| **TDI** | 入 | 数据输入（JTDI） | **PA15** |
| **TDO** | 出 | 数据输出（JTDO） | **PB3** |
| **nTRST** | 入（可选） | 测试复位 | **PB4** |
| **GND** | — | 共地（必须） | GND |
| **VTref / 3.3V** | — | 目标电平参考 / 供电相关 | 见仿真器说明 |

### 2.2 特点

- 线多、占脚多（完整 JTAG 用到 PA13/14/15、PB3/PB4 等）。  
- 战舰板载 **标准 20 针 JTAG 座**，可走满配 JTAG。  
- 正点原子与多数教程仍 **推荐用 SWD**（线少、省 IO，下载仿真同样够用）。

## 三、SWD 是什么

**SWD**（Serial Wire Debug，**串行线调试**）是 Arm 为 Cortex-M 推出的 **两线调试接口**：能力覆盖日常下载与断点调试，比 JTAG **少占很多 GPIO**。

### 3.1 信号线（入门必记）

| 信号 | MCU 脚（F103ZET6 默认） | 作用 |
|------|-------------------------|------|
| **SWDIO** | **PA13**（与 JTMS 同脚） | 双向数据 |
| **SWCLK** | **PA14**（与 JTCK 同脚） | 时钟 |
| **GND** | GND | 必须共地 |
| **NRST** | NRST | 建议接上，复位更稳 |
| **VTref** | 3.3V | 给仿真器作目标电平参考（20 针座常有） |

### 3.2 特点

- 核心只要 **SWDIO + SWCLK + GND**（建议再加 NRST、VTref）。  
- 与 JTAG **共用** PA13/PA14 等物理脚，协议不同。  
- Keil / CubeIDE + J-Link/ST-Link 对 STM32 **默认优先选 SW（SWD）**。  
- **不要把 PA13/PA14 当普通 GPIO 占用**，否则容易失去仿真下载能力。

## 四、JTAG 与 SWD 对照

| 项 | JTAG | SWD |
|----|------|-----|
| 全称 | Joint Test Action Group | Serial Wire Debug |
| 信号线（核心） | TCK、TMS、TDI、TDO（+可选 nTRST） | **SWCLK、SWDIO** |
| 占脚 | 多 | **少** |
| 战舰板 | 20 针座支持 | **推荐使用**（同一 20 针座） |
| 调试器设置 | Port = JTAG | Port = **SW** |

STM32 调试口常合称 **SWJ**（Serial Wire / JTAG），可配置为：

| 配置 | 结果 |
|------|------|
| 全功能 JTAG | PA13/14/15、PB3/4 等给调试 |
| **仅 SWD**（常用） | 保留 PA13/14；**释放** PA15、PB3、PB4 作 GPIO 等 |
| 关掉调试口 | 脚都可当 GPIO——**危险**：可能再也 SWD 不下程序 |

CubeMX：`SYS → Debug → Serial Wire`。  
库函数常见：`GPIO_PinRemapConfig(GPIO_Remap_SWJ_JTAGDisable, ENABLE)` → **关 JTAG、留 SWD**。

## 五、战舰板怎么接（仿真器 ↔ 开发板）

### 5.1 板载接口

战舰 **STM32F103ZET6** 一般带：

| 接口 | 说明 |
|------|------|
| **20 针 JTAG/SWD 座** | 接 J-Link / ST-Link 等；**同一插座**可按仿真器设置走 JTAG 或 SWD |
| **USB 转串口** | 多用于 `printf`，不是 SWD |
| **BOOT 跳线** | SWD 下载时保持从 Flash 启动（BOOT0=0）；仅串口 ISP 时改 BOOT |

具体针号、丝印以 **正点原子战舰原理图 / 用户手册** 为准（不同版次排版可能略有差异）。

### 5.2 SWD 最少接线（推荐）

| 仿真器侧（常见名） | 战舰 / MCU |
|--------------------|------------|
| SWDIO（TMS） | **PA13** |
| SWCLK（TCK） | **PA14** |
| GND | **GND** |
| RESET（建议） | **NRST** |
| VTref（20 针座） | **3.3V**（目标参考，按仿真器要求） |

```text
电脑 USB ──► J-Link / ST-Link ──SWD──► 战舰 20 针口（PA13/PA14）
                    │
                    └─ 下载程序 / 断点 / 读变量
```

### 5.3 使用注意

| 注意 | 说明 |
|------|------|
| 共地 | 仿真器与板子 **GND 必须相连** |
| 上电 | 战舰用适配器或 USB 供电时，确认电源开关已打开（若板载有开关） |
| 协议选择 | 只接 SWD 两根线时，Keil 里选 **SW**，不要选 JTAG |
| 线序 | 按 **功能名 / 手册针号** 对，勿只靠杜邦线颜色 |
| BOOT0 | **SWD 烧录**保持启动自 Flash（BOOT0=0）；串口 ISP 才临时改 1 |
| 工程器件 | Keil 选 **STM32F103ZE**（ZET6），勿再选 C8 |

### 5.4 与旧资料中「蓝 pill」的关系

仓库里部分早期文档仍以蓝 pill（F103C8）举例。自本文起，**调试与下载叙述以战舰 F103ZET6 为准**。  
芯片同属 F103，**SWD 仍是 PA13/PA14**；差别在封装引脚数、板载 20 针座与外设资源。

## 六、和「串口下载」的区别

| | SWD（仿真器） | 串口 ISP |
|--|---------------|----------|
| 接口 | SWDIO / SWCLK（战舰 20 针座） | USART1 等 + USB 转串口 |
| BOOT0 | 通常 **0** | 临时 **1**，下完改回 |
| 断点调试 | **可以** | 不可以 |
| 入门推荐 | **首选** | 备用 |

---

## 七、Keil 侧要点（战舰）

1. **Device**：STM32F103ZE / ZET6。  
2. **Debug**：J-Link 或 ST-Link；**Port = SW**（用 SWD 时）。  
3. 需要 PA15、PB3、PB4 作普通 IO 时：工程里 **禁用 JTAG、保留 SWD**。  
4. 烧录失败：先查供电开关、20 针线、GND、仿真器驱动，再查是否占用了 PA13/14。

## 八、常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 找不到仿真器 | 驱动/USB/线材 | 换口、装 J-Link/ST-Link 驱动 |
| Connect failed | 线序错、未共地、板未上电 | 对 SWDIO/SWCLK/GND；开电源 |
| 以前能下、现在不能 | PA13/14 被当 GPIO | 串口 ISP 救砖；改程序恢复 SWD |
| 选了 JTAG 连不上 | 实际按 SWD 接线 | 改选 **SW** |
| 烧录成功但程序不跑 | BOOT 在系统存储器档 | BOOT0 改回 0 后复位 |

## 九、自检

1. 战舰 MCU 型号是什么？SWD 两根信号脚是哪两个？  
   → **STM32F103ZET6**；**PA13（SWDIO）、PA14（SWCLK）**。  
2. 板载调试座通常是什么？推荐 JTAG 还是 SWD？  
   → **20 针 JTAG/SWD 座**；推荐 **SWD**。  
3. 为什么别占用 PA13/14？  
   → 占用后仿真器往往 **无法再下载/调试**。  
4. 仿真器和 USB 转串口各管什么？  
   → 仿真器：**烧录+调试**；USB 串口：**打印日志**。  
5. SWD 下载时 BOOT0 一般怎么接？  
   → **0**（从 Flash 启动）。
