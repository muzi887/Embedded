## 一、OLED 是什么

**OLED**（Organic Light-Emitting Diode，有机发光二极管）是一种 **自发光** 的显示技术：每个像素自己发光，不需要背光板，黑位真黑、对比度高、功耗低。

入门套件里常见的是 **0.96 英寸 OLED 模块**：


| 项        | 说明                                |
| -------- | --------------------------------- |
| **尺寸**   | 约 0.96 英寸（对角线）                    |
| **分辨率**  | 常见 **128 × 64** 像素（宽 128、高 64）    |
| **颜色**   | 单色（蓝/白/黄/蓝黄双色等，由模块面板决定）           |
| **驱动芯片** | 多为 **SSD1306**（也有 SH1106，库需匹配）    |
| **接口**   | 入门套件多为 **I2C 四线**；也有 SPI 七线版      |
| **供电**   | **3.3 V**（部分模块标 5 V 兼容，仍建议 3.3 V） |


**记忆口诀**：**OLED 自发光；128×64 小屏；SSD1306 管驱动；四线 I2C 最常见。**

### 1.1 和 LED 数码管的区别


|       | 板载 LED  | OLED 模块          |
| ----- | ------- | ---------------- |
| 能显示什么 | 亮 / 灭   | 字符、数字、简单图形       |
| 信息量   | 1 bit   | 整屏点阵             |
| 驱动方式  | 一根 GPIO | **I2C 总线** + 驱动库 |
| 典型用途  | 状态指示    | 温度、菜单、告警文字       |


### 1.2 在本项目中的角色

现场嵌入式设备常有一块 **机身小屏** 显示关键测值（温度、状态、告警）；OLED 就是入门阶段模拟这块小屏。复杂 UI 在手机 App，OLED 只负责本地简要显示。

## 二、硬件与接线

### 2.1 模块引脚（I2C 四线版）


| 模块丝印    | 接 STM32（蓝 pill 常见接法） | 说明                        |
| ------- | -------------------- | ------------------------- |
| **GND** | GND                  | 必须共地                      |
| **VCC** | 3.3 V                | 勿接 5 V（除非模块明确 5 V 兼容且已验证） |
| **SCL** | **PB6**（I2C1_SCL）    | 时钟线                       |
| **SDA** | **PB7**（I2C1_SDA）    | 数据线                       |


用 **公对母** 杜邦线连接；线色自定，建议固定：**红 VCC、黑 GND、黄 SCL、绿 SDA**。

### 2.2 I2C 从机地址

SSD1306 的 I2C 地址由模块 **SA0 / D/C 引脚** 硬件决定：


| 地址（7 位）  | 常见写法       | 说明          |
| -------- | ---------- | ----------- |
| **0x3C** | 写地址 `0x78` | 最常见，江协默认    |
| **0x3D** | 写地址 `0x7A` | SA0 接 VCC 时 |


屏不亮时，两个地址都试一次。

### 2.3 上拉电阻

I2C 是 **开漏总线**，SCL、SDA 必须有 **上拉电阻** 才能变高：

- 多数 OLED 模块 **板载已带上拉**（4.7 kΩ 常见）
- 若通信不稳定，可外接 4.7 kΩ：SCL→3.3 V、SDA→3.3 V

详见 [STM32-GPIO说明.md](./STM32-GPIO说明.md) §5.5（开漏与 I2C 关系）。

### 2.4 坐标系

SSD1306 显存按 **页（Page）** 组织：128 宽 × 8 页高 = 128×64 像素。

```text
原点 (0,0) 在左上角
  X →  0 … 127
Y
↓
0 … 63

一行 8 像素高 = 1 Page（Page0～Page7）
```

显示库里的 `OLED_ShowChar(x, y, ...)` 中，`y` 常以 **像素行** 或 **页** 为单位，以你用的库为准（江协库一般为 **像素坐标，y 每次 +16 换行**）。

## 三、OLED 如何与 MCU 通信

### 3.1 为什么用 I2C


| 对比   | I2C                        | SPI   |
| ---- | -------------------------- | ----- |
| 线数   | **2 根信号线**（SCL + SDA）+ 电源地 | 4 根以上 |
| 速度   | 100 kHz 够用（小屏刷新）           | 更快    |
| 入门模块 | **出厂即 I2C 四线**             | 较少    |


OLED 刷新整屏数据量不大，I2C **100 kHz** 标准模式足够。

### 3.2 I2C 两线含义


| 线       | 全称           | 作用              |
| ------- | ------------ | --------------- |
| **SCL** | Serial Clock | 时钟，由主机（STM32）产生 |
| **SDA** | Serial Data  | 数据，双向           |


通信过程：**主机发从机地址 → 发寄存器/命令 → 发显示数据**。

### 3.3 江协课程：软件 I2C vs 硬件 I2C


| 方式         | 做法                      | 特点                         |
| ---------- | ----------------------- | -------------------------- |
| **软件 I2C** | 用 GPIO 模拟 SCL/SDA 时序    | 江协 OLED 章节默认；引脚随意改         |
| **硬件 I2C** | 配置 I2C1 外设，PB6/PB7 复用开漏 | 省 CPU；需配 `GPIO_Mode_AF_OD` |


江协教程通常先做 **MyI2C**（软件模拟），再驱动 **OLED**；与本工程 Keil + SPL 路线一致。

### 3.4 软件 I2C 基本时序（了解）

```text
起始条件 → 发 7 位地址 + 读写位 → 应答 → 发数据/命令 → 停止条件
```

江协 `MyI2C` 模块典型函数：


| 函数                    | 作用                    |
| --------------------- | --------------------- |
| `MyI2C_Init()`        | 配置 SCL、SDA 为开漏输出，释放总线 |
| `MyI2C_Start()`       | 起始信号                  |
| `MyI2C_Stop()`        | 停止信号                  |
| `MyI2C_SendByte()`    | 发送 1 字节               |
| `MyI2C_ReceiveByte()` | 接收 1 字节（读操作时用）        |


OLED 写命令/数据时，在 I2C 帧里还要区分 **Co（控制/数据）位**，由 `OLED_W_SCL` / `OLED_W_SDA` 封装。

## 四、驱动代码结构（江协 / 本工程推荐）

跟完江协 OLED 章节后，工程里通常新增：

```text
project/
  Hardware/
    MyI2C.c / MyI2C.h      — 软件 I2C 底层
    OLED.c / OLED.h        — SSD1306 驱动与显示 API
    OLED_Font.h            — 字模（ASCII 6×8 / 8×16 等）
  User/
    main.c                 — OLED_Init(); OLED_ShowString(...);
```

**分层关系**：

```text
main.c
  └─ OLED_ShowString / ShowChar / ShowNum …
       └─ OLED_WriteCommand / OLED_WriteData
            └─ MyI2C_Start / SendByte / Stop
                 └─ GPIO 操作 SCL、SDA
```

## 五、OLED 常用 API 与显示流程

### 5.1 使用流程（四步）

```text
1. MyI2C_Init()     — 初始化 I2C 引脚
2. OLED_Init()      — 写 SSD1306 初始化命令序列
3. OLED_Clear()     — 清屏（可选）
4. OLED_ShowXxx()   — 显示字符/字符串/数字
5. OLED_Update()    — 部分库需手动刷新显存到屏幕（江协新版有显存缓冲）
```

具体是否有 `OLED_Update()` 取决于库实现；江协 **带显存** 的写法改完缓冲区后需 **Update** 才上屏。

### 5.2 常用显示函数（江协）


| 函数                                    | 作用      | 示例                                   |
| ------------------------------------- | ------- | ------------------------------------ |
| `OLED_Init()`                         | 初始化屏幕   | 上电调用一次                               |
| `OLED_Clear()`                        | 清空显存/屏幕 | 切换界面前                                |
| `OLED_ShowChar(x, y, char, size)`     | 显示单个字符  | `OLED_ShowChar(1, 1, 'A', 16)`       |
| `OLED_ShowString(x, y, string, size)` | 显示字符串   | `OLED_ShowString(1, 1, "Hello", 16)` |
| `OLED_ShowNum(x, y, num, len, size)`  | 显示无符号整数 | `OLED_ShowNum(1, 3, 123, 3, 16)`     |
| `OLED_ShowSignedNum(...)`             | 显示有符号数  | 温度负值                                 |
| `OLED_ShowHexNum(...)`                | 显示十六进制  | 调试寄存器                                |
| `OLED_ShowBinNum(...)`                | 显示二进制   | 教学演示                                 |
| `OLED_Update()`                       | 刷新到屏幕   | 改显存后调用                               |


`size` 常见 **12 / 16**（像素高度），需与 `OLED_Font.h` 字模匹配。

### 5.3 最小示例

```c
#include "MyI2C.h"
#include "OLED.h"

int main(void)
{
    MyI2C_Init();
    OLED_Init();

    OLED_ShowString(1, 1, "STM32", 16);
    OLED_ShowString(1, 17, "Temp: 25C", 16);
    OLED_Update();          // 若库带显存缓冲

    while (1)
    {
    }
}
```

### 5.4 SSD1306 命令 vs 数据


| 类型          | 含义     | 典型例子           |
| ----------- | ------ | -------------- |
| **Command** | 配置显示方式 | 开显示、设对比度、设寻址模式 |
| **Data**    | 写入显存像素 | 清屏、画点、写字后的像素数据 |


驱动层用 `OLED_WriteCommand()` / `OLED_WriteData()` 区分；应用层一般只调 `ShowString` 等。

## 六、与 GPIO 知识的联系


| OLED / I2C 现象   | 对应 GPIO 知识                                     |
| --------------- | ---------------------------------------------- |
| SCL、SDA 开漏 + 上拉 | [STM32-GPIO说明.md](./STM32-GPIO说明.md) §5.5 开漏输出 |
| 硬件 I2C 引脚模式     | `GPIO_Mode_AF_OD` 复用开漏                         |
| 软件 I2C 引脚模式     | `GPIO_Mode_Out_OD` 开漏输出 + 读 SDA 时切换输入          |
| 先开 GPIO 时钟      | 与 LED、Key 一样，`RCC_APB2PeriphClockCmd`          |

## 七、常见卡点


| 现象           | 可能原因                     | 处理                    |
| ------------ | ------------------------ | --------------------- |
| 屏完全不亮        | VCC/GND 未接、未共地           | 查 3.3 V 与 GND         |
| 屏亮但无内容       | 未 `OLED_Init`、未 `Update` | 确认初始化与刷新              |
| 全白 / 全黑 / 花屏 | I2C 地址错                  | 试 **0x3C** / **0x3D** |
| 无反应          | SCL、SDA 接反或未接            | 对调检查 PB6/PB7          |
| 偶尔乱码         | 杜邦线接触不良                  | 换线、插紧                 |
| 编译通过但不显示     | 字模 `size` 与函数不一致         | 统一用 16 或 12           |
| 与 SH1106 混淆  | 驱动芯片不同                   | 换 SH1106 专用库或确认模块型号   |

## 八、自检

1. OLED 模块 I2C 版有几根线？分别叫什么？
  → **4 根**：GND、VCC、SCL、SDA。
2. 入门套件 SSD1306 常见 I2C 地址是多少？
  → **0x3C**（备选 0x3D）。
3. 蓝 pill 接 OLED 时 SCL、SDA 常用哪两个脚？
  → **PB6（SCL）、PB7（SDA）**。
4. I2C 为什么用开漏而不是推挽？
  → 多设备共线，**线与**；高电平靠 **上拉**，避免冲突。
5. `OLED_ShowString` 和 `MyI2C_SendByte` 谁在上层？
  → **ShowString** 在上层；SendByte 是最底层发字节。
6. 128×64 分辨率中，「128」指宽还是高？
  → **宽**（X 方向 0～127）。
