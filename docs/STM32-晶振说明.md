## 一、晶振是什么

**晶振（Crystal Oscillator / 石英晶体谐振器）**：利用石英晶体的压电效应，在电路配合下产生 **稳定频率** 的时钟信号，供 MCU、通信模块等「打拍子」。

可以把它理解成数字系统的 **节拍器**：CPU 取指、定时器计数、串口比特时间，都建立在某个基准频率之上。


| 相关称呼                 | 说明                                           |
| -------------------- | -------------------------------------------- |
| **无源晶振**（晶体、Crystal） | 两脚或四脚晶体本身；需 MCU 内部振荡电路 + 负载电容才能起振            |
| **有源晶振**（Oscillator） | 模块内部已含振荡电路，输出方波时钟，直接接到 MCU 的 OSC_IN（或 MCO 等） |
| **陶瓷谐振器**            | 便宜、精度低于石英晶振；部分低成本板使用                         |


蓝 pill 上那颗银色椭圆件，一般是 **无源 8 MHz 晶振**（HSE）。

![[attachments/stm32-blue-pill-board.png]]

**口诀**：**晶振给频率，RCC 做分配，PLL 能倍频；没稳时钟，定时和波特率都对不齐。**

## 二、STM32 里和晶振相关的时钟源

F103 片内 **RCC** 管理多路时钟源。和「晶振」直接相关的是外部高速 / 低速晶振；片内还有 RC 振荡器作备用。


| 名称      | 英文                  | 典型频率                | 来源            | 主要用途                     |
| ------- | ------------------- | ------------------- | ------------- | ------------------------ |
| **HSE** | High Speed External | 蓝 pill 常见 **8 MHz** | 外部高速晶振（或有源时钟） | 经 PLL 得到 **72 MHz** 系统时钟 |
| **HSI** | High Speed Internal | **8 MHz**           | 片内 RC         | 上电默认；HSE 失败时可回退          |
| **LSE** | Low Speed External  | **32.768 kHz**      | 外部低速晶振        | **RTC** 日历时钟、低功耗计时       |
| **LSI** | Low Speed Internal  | 约 **40 kHz**        | 片内 RC         | 独立看门狗、粗略低速时钟             |


```text
外部 8 MHz 晶振 ──► HSE ──► PLL（×9 等）──► SYSCLK 72 MHz
                                              ├── AHB / APB ──► CPU、外设
                                              └── 定时器、USART 波特率等

外部 32.768 kHz ──► LSE ──► RTC
片内 HSI / LSI ──► 备用或特殊外设
```

**HSE 与定时器「外部时钟」不是一回事**：  
HSE 是 **系统主时钟源**；定时器 ETR「外部时钟」是 **GPIO 上的脉冲当计数拍**（见定时器文档）。

## 三、蓝 pill 上的晶振


| 器件           | 频率             | 接芯片脚                                     | 说明                 |
| ------------ | -------------- | ---------------------------------------- | ------------------ |
| 板载主晶振        | **8 MHz**      | **PD0 / OSC_IN**、**PD1 / OSC_OUT**       | HSE；经 PLL 到 72 MHz |
| RTC 晶振（部分板有） | **32.768 kHz** | **PC14 / OSC32_IN**、**PC15 / OSC32_OUT** | LSE；无贴装时这两脚可作 GPIO |


旁路通常还有 **负载电容**（几十 pF 量级，以原理图为准），与晶体匹配才能稳定起振。入门阶段 **不必改焊**；换晶振频率时必须同步改软件里的 `HSE_VALUE` 与 PLL 配置。

## 四、从晶振到 72 MHz

江协 / 标准库常见目标：**系统时钟 SYSCLK = 72 MHz**。

```text
8 MHz（HSE）
    → PLL 倍频（常见 ×9）
    → SYSCLK = 72 MHz
    → AHB 预分频（常 /1）→ HCLK = 72 MHz（给内核等）
    → APB1 预分频（常 /2）→ PCLK1 = 36 MHz（注意：部分定时器时钟有 ×2 规则）
    → APB2 预分频（常 /1）→ PCLK2 = 72 MHz（USART1 等）
```


| 步骤                   | 谁负责                                                    |
| -------------------- | ------------------------------------------------------ |
| 硬件起振                 | 晶振 + 负载电容 + MCU 振荡电路                                   |
| 选 HSE、配 PLL、切 SYSCLK | `SystemInit()` / `system_stm32f10x.c` 或 CubeMX Clock 页 |
| 外设门控时钟               | `RCC_APBxPeriphClockCmd` 等（每个外设还要单独开）                  |


工程里 `HSE_VALUE`（常见在 `stm32f10x.h`）必须与 **实际晶振频率** 一致；若写成 8 MHz 而板上是 12 MHz，则串口波特率、延时、定时周期会整体偏差。

复位后默认往往先跑 **HSI**，`SystemInit()` 再切到 **HSE+PLL**；若 HSE 起振失败，可能仍停在 HSI，表现为「能跑但波特率不对」。

## 五、为什么晶振重要


| 现象层                | 和时钟的关系                                        |
| ------------------ | --------------------------------------------- |
| **串口波特率**          | 比特时间由外设时钟分频得到；主频错 → 乱码                        |
| **定时器周期**          | `T = (PSC+1)(ARR+1)/f_CK` 里的 `f_CK` 来自系统/总线时钟 |
| `**Delay_ms` 类延时** | 若按 72 MHz 校准而实际不是，延时会偏                        |
| **RTC 走时**         | 依赖 LSE 32.768 kHz 是否存在且稳定                     |
| **USB（进阶）**        | 对时钟精度要求更高，通常必须可靠 HSE                          |


入门可以暂时「默认板子已配好 72 MHz」，但要知道：**晶振是整条时钟链的源头之一**。

## 六、无源晶振使用要点

1. **两脚接 OSC_IN / OSC_OUT**，不要当普通 GPIO 乱接。
2. **负载电容** 按晶体手册与板厂设计匹配；自己飞线外接晶振时布线尽量短。
3. 换频率（如 8→12 MHz）必须改 `**HSE_VALUE` + PLL 倍频**，使 SYSCLK 仍落在芯片允许范围（F103 常用上限 72 MHz）。
4. **有源晶振** 一般只驱动 OSC_IN，OSC_OUT 脚按手册处理（常悬空或按应用笔记）。
5. 测量可用示波器看 OSC 脚（注意探头负载可能影响起振）；软件侧可先查串口波特率是否准，间接判断主频。

## 七、与本工程 / 套件对照


| 项    | 本套件常见情况                                      |
| ---- | -------------------------------------------- |
| 硬件   | 蓝 pill **8 MHz HSE** 已焊好                     |
| 软件   | `system_stm32f10x.c` 中配置到 **72 MHz**（以工程宏为准） |
| 你要做的 | 入门阶段通常 **不用手改晶振**；改时钟配置时核对 `HSE_VALUE`       |
| 相关实验 | 串口 9600/115200、定时器 1 ms 节拍，都默认这条时钟链正确        |

## 八、常见问题


| 现象             | 可能原因                    | 处理                     |
| -------------- | ----------------------- | ---------------------- |
| 串口固定比例乱码       | 实际主频与假定不符；HSE 未起来仍用 HSI | 查 `HSE_VALUE`、PLL；确认起振 |
| 定时不准           | 同上；或定时器时钟源搞错            | 先确认 SYSCLK；再查 PSC/ARR  |
| 刚上电能跑、时钟相关全错   | 仍停在 HSI 8 MHz           | 调试读 RCC 时钟源寄存器；查晶振焊接   |
| 想用 RTC 但走时飞    | 无 LSE 或未使能              | 确认板载 32.768 kHz；配置 LSE |
| 自己换了 12 MHz 晶振 | 仍按 8 MHz 配 PLL          | 更新 `HSE_VALUE` 与倍频系数   |

## 九、自检

1. 蓝 pill 主晶振常见频率是多少？接到哪两个脚？
  → **8 MHz**；**PD0/PD1（OSC_IN/OUT）**。
2. HSE 和 HSI 的区别？
  → HSE 外部晶振（准）；HSI 片内 RC（上电默认、精度较差）。
3. 8 MHz 如何变成 72 MHz？
  → **PLL 倍频**（常见 ×9）。
4. 32.768 kHz 晶振主要给谁用？
  → **LSE / RTC**。
5. 为什么换晶振频率要改软件？
  → 波特率、延时、定时都按 `HSE_VALUE` 与 PLL 结果计算。
