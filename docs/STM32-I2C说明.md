## 一、I2C 是什么；I2C1 / I2C2 是什么

### 1.1 I2C（协议 / 总线）

**I2C**（Inter-Integrated Circuit，集成电路间总线）是一种 **两线串行总线**：

| 项 | 说明 |
|----|------|
| **信号线** | **SCL**（Serial Clock，时钟）+ **SDA**（Serial Data，数据） |
| **电源地** | 模块上常再加 VCC、GND → **四线** |
| **拓扑** | 一根 SCL、一根 SDA 可挂 **多个从机**，靠 **7 位地址** 区分 |
| **双工** | **半双工**（SDA 同一时刻只有一个方向） |
| **时钟** | **有 SCL**（同步）；主机产生节拍，从机可时钟拉伸 |
| **电气** | **开漏总线**，SCL/SDA 需 **上拉** 才能变高 |
| **速率** | 标准约 **100 kHz**；快速约 **400 kHz**；杜邦线宜短 |

与 USART / SPI 对照：

| 名称 | 引脚 | 双工 | 时钟 | 设备 |
|------|------|------|------|------|
| **USART** | TX、RX | 全双工 | 异步（无时钟线） | 点对点 |
| **I2C** | SCL、SDA | 半双工 | 同步 | 多从机 |
| **SPI** | SCLK、MOSI、MISO、CS | 全双工 | 同步 | 多设备（各需 CS） |

**记忆**：I2C = **共时钟、共数据、靠地址认人**；少线、适合短距小板（OLED、EEPROM、传感器）。

### 1.2 I2C1 / I2C2（片上外设）

**I2C1、I2C2** 是 STM32 芯片里的 **两个独立 I2C 硬件控制器**（外设实例），都能跑同一套 I2C 协议，只是 **挂在不同引脚、不同 APB 时钟** 上。

F103C8T6（蓝 pill）常见默认脚：

| 外设 | 默认 SCL / SDA | 本套件用途 |
|------|----------------|------------|
| **I2C1** | **PB6 / PB7** | ★ OLED（入门默认） |
| **I2C2** | **PB10 / PB11** | 第二路 I2C（另挂传感器/EEPROM 等） |

补充：

- I2C1 还可 **重映射** 到 PB8/PB9（见引脚说明）。
- 口语「开 I2C」可指协议，也可指使能 `I2C1`/`I2C2` 外设。
- 本工程 OLED 多用 **软件模拟 I2C**（GPIO 开漏 bit-bang），不强制开硬件 `I2C1`；硬件 I2C 时引脚配 **`GPIO_Mode_AF_OD`**。

```text
协议层：I2C（SCL + SDA + 地址 + 时序）
硬件层：STM32 的 I2C1、I2C2（两套控制器，可同时各挂一串总线）
软件层：软件 I2C（GPIO 模拟）或 硬件 I2C（外设寄存器 / HAL）
```

## 二、SCL、SDA 与开漏

| 缩写 | 全称 | 作用 |
|------|------|------|
| **SCL** | Serial Clock Line | 主机发时钟；从机可拉低做时钟拉伸 |
| **SDA** | Serial Data Line | 传地址、命令、数据；双向 |

**必须用开漏**（`Out_OD` / `AF_OD`）的原因：多设备共线，只能 **线与**——谁都能拉低，高电平靠上拉；若用推挽，一台拉高、一台拉低会短路。

```text
        3.3V
          │
       [上拉 R]  ← 常见 4.7 kΩ（OLED 模块常已板载）
          │
    ──────●────── SCL 或 SDA
         /|\
     MCU OLED 传感器 …（均为开漏）
```

详见 [STM32-GPIO说明.md](./STM32-GPIO说明.md) §5.5、[入门接线与线材说明.md](./入门接线与线材说明.md) §7。

## 三、总线符号与基本条件

| 符号 | 名称 | 电平动作（开漏） |
|------|------|------------------|
| **S** | START 起始 | SCL=1 时，SDA **高→低** |
| **Sr** | Repeated START 重复起始 | **不先 STOP**，再做一次「SCL=1 时 SDA 高→低」 |
| **P** | STOP 停止 | SCL=1 时，SDA **低→高** |
| **A** | ACK 应答 | 接收方在第 9 个时钟把 SDA **拉低** |
| **N** | NACK 不应答 | 接收方释放 SDA，保持 **高** |
| **W / R** | 读写位 | 地址字节最低位：0=写，1=读 |

**Sr 用途**：指定地址读时，假写设完内部指针后，用 Sr 切到读，**不断开总线**，指针不会因 STOP 而乱。

两个「地址」勿混：

| 名称 | 含义 | 例子 |
|------|------|------|
| **I2C 从机地址** | 总线上认哪个芯片 | OLED `0x3C` → 写出常为 `0x78`（`<<1` 后加 W） |
| **内部地址（字地址）** | 芯片内部第几单元/寄存器 | EEPROM `0x10`；传感器某寄存器 |

## 四、发送 / 接收一个字节的时序

共通规则：

1. **高位先发（MSB first）**：先 bit7，再 bit6 … 到 bit0  
2. **SCL 低改数据，SCL 高采样**  
3. **1 字节 = 8 数据位 + 1 应答位**（共 9 个 SCL 脉冲）

### 4.1 发送一个字节（主机 → 从机）

主机驱动 SDA 发 8 位；第 9 拍由 **从机** 拉低 = ACK。

```text
SCL:  低改数据，高采样
      ┌─┐ ┌─┐ … ┌─┐ ┌─┐
      └─┘ └─┘   └─┘ └─┘
       D7  D6 …  D0  ACK
      ←── 主机驱动 ──→ ←从机→
```

过程：

```text
for i = 7 downto 0:
    SCL = 0
    SDA = Byte 的第 i 位
    SCL = 1          // 从机采样
    SCL = 0

SCL = 0
主机释放 SDA
SCL = 1
读 SDA：0=ACK，1=NACK
SCL = 0
```

本工程写屏（不读 ACK，仅占第 9 时钟）：

```192:208:project/Hardware/OLED.c
void OLED_I2C_SendByte(uint8_t Byte)
{
	uint8_t i;
	for (i = 0; i < 8; i++)
	{
		OLED_W_SDA(!!(Byte & (0x80 >> i)));
		OLED_W_SCL(1);
		OLED_W_SCL(0);
	}
	OLED_W_SCL(1);		// 额外一个时钟（此处未读 ACK）
	OLED_W_SCL(0);
}
```

### 4.2 接收一个字节（从机 → 主机）

主机只发时钟；**从机驱动** 前 8 位；第 9 拍由 **主机** 发 ACK/NACK。

```text
      D7 … D0  ACK/NACK
      ←从机→  ←主机决定→
```

过程：

```text
主机释放 SDA
for i = 7 downto 0:
    SCL = 0 → SCL = 1 → 读 SDA 拼入 Byte → SCL = 0

SCL = 0
若还要继续读：SDA = 0（ACK）
若是最后一字节：SDA = 1（NACK）
SCL = 1 → SCL = 0
释放 SDA
```

| 第 9 位 | 含义 |
|---------|------|
| **ACK** | 收到了，请再发下一字节 |
| **NACK** | 读完了，停发 → 随后通常 **STOP** |

口诀：**发：主机摆 8 位，从机第 9 拍点头；收：从机摆 8 位，主机第 9 拍决定还要不要。**

## 五、四种访问模式（+ 多字节）

以下以 **EEPROM 类从机**（有内部地址指针）为模型；OLED 写命令/数据形态接近「指定地址写」。

### 5.1 当前地址写

不指定内部地址，往 **当前指针** 写；写完指针通常 **+1**。

```text
单字节：
S ─ [Dev+W] A ─ [Data] A ─ P

多字节（连续写）：
S ─ [Dev+W] A ─ [D0] A ─ [D1] A ─ … ─ [Dn] A ─ P
```

适用：指针已在正确位置，继续写下一单元。

### 5.2 指定地址写

先发 **内部地址**，再写数据（随机写 / 带字地址写）。

```text
单字节：
S ─ [Dev+W] A ─ [WordAddr] A ─ [Data] A ─ P

多字节（从 WordAddr 起连续写）：
S ─ [Dev+W] A ─ [WordAddr] A ─ [D0] A ─ [D1] A ─ … ─ [Dn] A ─ P
```

OLED 写命令（控制字节 `0x00` 类似「指定写命令」）：

```text
S ─ 0x78 ─ 0x00 ─ Command ─ P
```

见 `OLED_WriteCommand` / `OLED_WriteData`。

**多字节注意（EEPROM）**：页写有页边界限制（如 8/16 字节一页），跨页需拆分或另开写周期。

### 5.3 当前地址读

不指定内部地址，直接读 **当前指针**。

```text
单字节：
S ─ [Dev+R] A ─ [Data] N ─ P

多字节：
S ─ [Dev+R] A ─ [D0] A ─ [D1] A ─ … ─ [Dn] N ─ P
                 ↑ ACK 继续     ↑ 最后一字节 NACK
```

适用：刚设过指针或刚写过，指针已对齐。

### 5.4 指定地址读（最常用）

I2C 读帧里 **放不下内部地址**，故：

1. **假写**：用写方向发 `[WordAddr]`，只设指针、不写数据  
2. **Sr**：不 STOP，重复起始  
3. **真读**：`[Dev+R]` 后收数据  

```text
单字节：
S ─ [Dev+W] A ─ [WordAddr] A ─ Sr ─ [Dev+R] A ─ [Data] N ─ P
     ↑ 假写设指针              ↑ 重复起始         ↑ 真读

多字节：
S ─ [Dev+W] A ─ [WordAddr] A ─ Sr ─ [Dev+R] A
                              │
                    [D0] A ─ [D1] A ─ … ─ [Dn] N ─ P
```

为何必须 **Sr** 而不能 STOP 再 START：

| 方式 | 结果 |
|------|------|
| **Sr** | 总线不断，指针仍停在 WordAddr ✅ |
| STOP 再 S | 部分芯片指针复位/不确定 ❌ |

`HAL_I2C_Mem_Read()` 底层即此流程。

### 5.5 四种模式对照

| 模式 | 指定内部地址 | START | 方向 | 典型用途 |
|------|--------------|-------|------|----------|
| 当前地址写 | 否 | 1×S | 全程写 | 连续写下一字节 |
| 指定地址写 | 是 | 1×S | 全程写 | 写某寄存器/单元 |
| 当前地址读 | 否 | 1×S | 全程读 | 指针已在位 |
| 指定地址读 | 是（假写） | S + **Sr** | 先写后读 | **读任意位置** |

多字节统一规则：

> **中间字节 ACK，最后一字节 NACK（读时），然后 STOP。**  
> 写时每个数据字节通常都要从机 ACK。

## 六、知识怎么串起来

```text
I2C 协议
 ├─ 电气：开漏 + 上拉（SCL/SDA）
 ├─ 条件：S / Sr / P
 ├─ 字节：MSB、8+1（ACK/NACK）
 └─ 访问：当前/指定 × 读/写（指定读 = 假写 + Sr + 读）
        └─ 多字节：指针自增；读末字节 NACK

STM32
 ├─ I2C1（PB6/7）— 入门 OLED
 └─ I2C2（PB10/11）— 第二路总线
```

## 七、常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 完全无应答 | 无上拉、地址错、未共地 | 查 4.7 kΩ；扫 0x3C/0x3D；GND |
| 指定地址读错数据 | 中间用了 STOP | 改用 **Sr** |
| 读多字节卡住 | 末字节仍发 ACK | 最后一字节改 **NACK** 再 STOP |
| OLED 不亮 | SCL/SDA 对调、电平 | 对调检查；3.3 V |
| 软件 I2C 偶发错 | 线过长、无延时 | 缩短杜邦线；位间稍延时 |

## 八、自检

1. I2C 和 I2C1/I2C2 分别指什么？  
   → **I2C**：两线协议；**I2C1/2**：片上两套硬件控制器。  
2. SCL、SDA 为何必须开漏？  
   → 多机共线 **线与**，高靠上拉，避免推挽对掐。  
3. 发一字节谁回 ACK？收一字节谁发 ACK/NACK？  
   → 发：从机 ACK；收：主机 ACK/NACK。  
4. Sr 是什么？指定地址读为何用它？  
   → **Repeated START**；假写设指针后不断总线再切读。  
5. 多字节读时中间与最后应答有何不同？  
   → 中间 **ACK**，最后 **NACK**，然后 **STOP**。
