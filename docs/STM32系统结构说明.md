## 一、一句话理解

下图是 **STM32（以 F1 系列 / Cortex-M3 为代表）的系统结构**：  
**CPU 内核** 通过 **三条总线** 取指令、读数据，经 **总线矩阵** 访问 **Flash、SRAM** 和挂在 **AHB / APB** 上的 **外设**；**DMA** 可在不经 CPU 的情况下搬运数据。

![[attachments/stm32-system-architecture.png]]

**记忆口诀**：**ICode 取指令，DCode 读常量，System 跑外设；矩阵做调度，APB 挂串口定时器。**

---

## 二、读图顺序（图中红圈区域）

建议按数据流 **从内核向外** 理解：

```text
① Cortex-M3 内核 + NVIC
② ICode / DCode / System 三条总线
③ 总线矩阵（仲裁）
④ Flash、SRAM、DMA、RCC
⑤ AHB 系统总线
⑥ 桥接1 / 桥接2
⑦ APB2（高速外设）、APB1（低速外设）
⑧ 各外设模块（GPIO、USART、TIM、ADC …）
```

## 三、Cortex-M3 内核

| 项 | 说明 |
|----|------|
| **是什么** | Arm **Cortex-M3** 处理器核，执行 Thumb-2 指令 |
| **在图中位置** | 左上红圈 |
| **片上邻居** | 内置 **NVIC**（中断控制器）、**SysTick**（系统节拍） |
| **蓝 pill** | STM32F103C8T6 即此内核 + ST 外设 |

CPU 本身 **不直接连引脚**；访问 GPIO、UART 等必须经 **System 总线 → 总线矩阵 → AHB/APB**。

详见 [ARM处理器系列说明.md](./ARM处理器系列说明.md)。

## 四、三条内核总线

图中内核右侧 **三条并行总线**（红圈）分工固定：

| 总线 | 英文 | 方向 | 主要访问对象 | 典型内容 |
|------|------|------|--------------|----------|
| **ICode** | Instruction Code | 内核 → Flash | **取指令** | `main()` 里的机器码 |
| **DCode** | Data Code | 内核 ↔ Flash | **读常量** | `const` 表、字符串字面量 |
| **System** | System Bus | 内核 ↔ 矩阵 | **通用访问** | SRAM、外设寄存器、堆栈 |

```text
         ┌── ICode ──► Flash（指令）
Cortex-M3├── DCode ──► Flash（只读数据）
         └── System ─► 总线矩阵 ─► SRAM / 外设 / DMA
```

**为何要分三条**：取指与读数据可 **并行**，提高吞吐；这是理解「Flash 等待周期」「从 Flash 跑代码比 RAM 慢」的基础。

---

## 五、总线矩阵（Bus Matrix）

| 项 | 说明 |
|----|------|
| **是什么** | 片上 **交叉开关 / 仲裁器**，多主设备访问多从设备时 **分配总线使用权** |
| **主设备（Master）** | Cortex-M3（System）、**DMA1**、**DMA2**、以太网 MAC 等（视型号） |
| **从设备（Slave）** | Flash 接口、SRAM、AHB 外设、APB 桥 |
| **你会遇到的现象** | DMA 搬 ADC 时 CPU 仍可访问别的外设；极端同时访问会 **等待一个周期** |

**记忆**：矩阵 = **交通枢纽**；谁都能过，但要 **排队拿路权**。

## 六、存储器与扩展接口

### 6.1 Flash

| 项 | 说明 |
|----|------|
| **作用** | 存 **程序代码**、常量；上电从 Flash 启动 |
| **连接** | 经 **Flash 接口** 接 ICode、DCode |
| **蓝 pill** | 常见 **64 KB**（部分标 128 KB） |

### 6.2 SRAM（红圈）

| 项 | 说明 |
|----|------|
| **作用** | **运行时变量**、堆栈、缓冲区 |
| **连接** | 经总线矩阵接 System 总线 |
| **蓝 pill** | **20 KB** |
| **速度** | 比 Flash 快，无等待周期 |

### 6.3 FSMC（红圈，部分 F103 型号）

| 项 | 说明 |
|----|------|
| **是什么** | **Flexible Static Memory Controller**，扩展外部并行存储器 |
| **用途** | 外接 SRAM、NOR Flash、部分 **TFT LCD** |
| **蓝 pill C8** | **无 FSMC**；大图含 FSMC 为 **高密度 F103** 或系列通用框图 |

### 6.4 SDIO（红圈，部分型号）

| 项 | 说明 |
|----|------|
| **是什么** | SD / MMC 卡主机接口 |
| **蓝 pill C8** | **无 SDIO**；F103 大容量或 F4 等才有 |

> 读图时：框图常画 **全系列能力**；以你芯片 **Datasheet 外设表** 为准。

## 七、DMA1 / DMA2

| 项 | 说明 |
|----|------|
| **位置** | 图左侧，各有多条 **通道** |
| **DMA1** | 图中 **7 个通道** |
| **DMA2** | 图中 **5 个通道** |
| **作用** | 外设 ↔ 内存 **自动搬数据**，减轻 CPU |
| **触发** | 外设发 **DMA 请求**（如 ADC 转换完成、UART RX 非空） |
| **典型用法** | ADC 连续采样进数组；UART 环形接收 |

```text
外设 ──DMA 请求──► DMA 控制器 ──总线矩阵──► SRAM 缓冲区
                      ▲
                 CPU 可去做别的事
```

详见 [STM32外设说明.md](./STM32外设说明.md) §6.3；专文：[STM32-DMA说明.md](./STM32-DMA说明.md)。

## 八、RCC 与 AHB 系统总线

### 8.1 RCC（红圈）

| 项 | 说明 |
|----|------|
| **是什么** | **Reset and Clock Control** |
| **作用** | 系统 / 外设 **时钟使能**、复位、PLL 倍频 |
| **为何重要** | 外设寄存器默认 **无时钟不工作**；Cube 生成 `__HAL_RCC_*_CLK_ENABLE()` |

### 8.2 AHB 系统总线（红圈）

| 项 | 说明 |
|----|------|
| **是什么** | **Advanced High-performance Bus**，高速主干 |
| **挂接** | DMA、SRAM、Flash 接口、**RCC**、**桥接1/2**、部分高速外设 |
| **时钟** | F103 上 AHB 最高 **72 MHz**（与 HCLK 同频） |

---

## 九、桥接与 APB 外设总线

AHB 速度高，大量外设跑较低频率；经 **桥接（Bridge）** 分到 **APB**：

```text
总线矩阵 ── AHB ──► 桥接1 ──► APB2（较快）
                 └── 桥接2 ──► APB1（较慢）
```

| 总线 | 相对速度 | 图中典型外设 |
|------|----------|--------------|
| **APB2**（红圈） | 较高（常 = HCLK 或 HCLK/2） | **GPIOA～G**、**AFIO**、**USART1**、**SPI1**、**TIM1/8**、**ADC1/2/3** |
| **APB1**（红圈） | 较低（常 = HCLK/2） | **USART2/3**、**UART4/5**、**I2C1/2**、**SPI2/3**、**TIM2～7**、**bxCAN**、**USB**、**DAC**、**PWR**、**BKP**、**IWDG/WWDG**、**RTC** |

### 9.1 与入门套件的对应

| 实验硬件 | 常用外设 | 所在总线 |
|----------|----------|----------|
| 板载 LED / 蜂鸣器 | GPIO | **APB2** |
| USB-TTL `printf` | **USART1**（PA9/10） | **APB2** |
| OLED | **I2C1**（PB6/7） | **APB1** |
| Wi-Fi/4G/BLE 模组 | USART2/3 等 | 多为 **APB1** |
| 舵机 PWM | TIM2/3/4 | **APB1** |
| 光敏 AO | ADC1 | **APB2** |

**实践提示**：USART1 在 APB2 上，波特率计算用 **PCLK2**；USART2 在 APB1 上，用 **PCLK1**——CubeMX 会自动算，手算时钟时要分清。

## 十二、一次「串口发字节」的数据路径

以 `HAL_UART_Transmit` 发送 `'A'` 为例，帮助把框图串起来：

```text
1. CPU 从 Flash（ICode）取指令
2. CPU 经 System 总线写 USART1 数据寄存器 DR
3. 总线矩阵仲裁 System 主设备
4. 经 AHB → 桥接1 → APB2 → USART1
5. USART1 移位从 TX 引脚发出位流
6. 若开 DMA：内存缓冲区经 DMA1 通道自动写 DR，CPU 不参与逐字节写
```

## 十三、与 STM32F103C8（蓝 pill）的差异说明

教学框图往往画 **整个 F1 系列** 能力；**C8** 是 **中等容量** 型号：

| 项目 | 框图可能画出 | F103C8 典型 |
|------|--------------|-------------|
| Flash | — | 64 KB |
| SRAM | — | 20 KB |
| ADC | ADC1/2/3 | **ADC1/2** |
| USART | 1/2/3 + UART4/5 | **1/2/3**，无 UART4/5 |
| FSMC / SDIO | 有 | **无** |
| DAC | 有 | **无** |
| GPIO | A～G | **A/B/C** 等部分端口 |

**学习方法**：框图理解 **架构不变**；具体有没有某外设查 [Datasheet](https://www.st.com/resource/en/datasheet/stm32f103c8.pdf) 的 **Device features** 表。

## 十四、与本项目的关系

| 结构层 | 本项目设备中的体现 |
|--------|-------------------|
| Cortex-M3 + Flash/SRAM | 跑采集、协议、状态机固件 |
| USART（APB1/2） | 连 **Wi-Fi/4G MQTT**、**BLE 透传**、调试串口 |
| I2C（APB1） | **OLED** 机身显示 |
| GPIO + TIM（APB2/1） | 告警、继电器、**舵机 PWM** |
| ADC（APB2） | 水位/压力等模拟量 |
| DMA | 高频采样、串口缓冲（量产优化） |
| IWDG | 野外 **7×24** 防死机 |
| RCC | 72 MHz 主频、外设时钟门控 |

软件分层仍见 [STM32外设说明.md](./STM32外设说明.md)；本文回答 **硬件在芯片里怎么连在一起**。

## 十五、自检问答

1. ICode 与 System 总线分别访问谁？  
   → ICode **取指 Flash**；System **访问 SRAM 与外设**。  

2. USART1 和 I2C1 在哪条 APB 上？  
   → USART1 在 **APB2**；I2C1 在 **APB1**。  

3. DMA 存在的意义？  
   → **硬件搬数据**，CPU 不必每个字节轮询。  

4. 为何要先开 RCC 时钟？  
   → 外设逻辑 **无时钟则寄存器不工作**。  

5. 蓝 pill 没有 FSMC，图里为什么有？  
   → 图为 **系列结构**；以具体型号手册为准。
