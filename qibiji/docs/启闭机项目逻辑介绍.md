# 启闭机项目逻辑介绍

本文档从系统视角说明 `qibiji` 启闭机固件的业务逻辑与模块协作关系，便于新人快速建立整体认识。更细的专题（开度统计、LCD 刷新、事件上报等）见 `qibiji/` 目录下既有 Markdown。

---

## 1. 项目是什么

本工程是运行在 **STM32F10x** 上的闸门启闭机控制器固件（Keil 工程名约 `SRTUV10`）。

设备职责：

- **驱动机门**：通过继电器控制电机上升 / 下降 / 停止
- **计量开度**：光电编码器脉冲 + 上/下限位 → 实时开度 0%～100%
- **本地人机**：五键 + LCD12864 查看/设置参数
- **远程上云**：4G 模块（UART2）经 MQTT/JSON 收发控制与属性/事件
- **近场配置**：UART3 近场 20B 帧协议（SBF-20）读写采样/上报周期等
- **参数持久化**：AT24C02 EEPROM 保存开度、圈数、报警阈值等

供电侧预留电池 / 太阳能等遥测寄存器；RS485 每秒轮询，当前解析多为预留。

## 2. 软件分层总览

```
┌─────────────────────────────────────────────────────────────┐
│  通信层：GPRS/MQTT JSON、SBF-20 近场、RS485                 │
├─────────────────────────────────────────────────────────────┤
│  人机层：按键 KEY_Process、液晶 Lcd_Process                 │
├─────────────────────────────────────────────────────────────┤
│  控制层：Dcmotor_Process（状态机汇合本地键与远程目标）       │
├─────────────────────────────────────────────────────────────┤
│  传感/时基层：TIM2 中断（编码器、限位、开度固化）+ ADC      │
├─────────────────────────────────────────────────────────────┤
│  数据层：armReg + Num_count / FLAG_* + EEPROM               │
└─────────────────────────────────────────────────────────────┘
```

**业务真相源**：`Register armReg`（`HARDWARE/REG/reg.h`）与运行时计数 `Num_count`、限位标志 `FLAG_OPENALL` / `FLAG_CLOSEALL`。  
本地按键与云端 `setting` 最终都写入同一套电机状态，再由继电器输出。

## 3. 启动与主循环

入口：`qibiji/USER/main.c`。

### 3.1 初始化顺序（摘要）

1. 延时、NVIC、独立看门狗  
2. 串口：UART1 调试(460800)、UART2 4G(115200)、UART3 近场(9600)、UART4 RS485(9600)  
3. LED / 按键 / 电机继电器 / ADC / LCD / 内存 / 4G / 开关量  
4. EEPROM `para_read` → `initReg`  
5. DS1302 时钟；TIM2（5ms 时基）、TIM4  

### 3.2 主循环任务

```c
while (1) {
  IWDG_Feed();
  sbf20_poll();            // 近场传输层轮询
  KEY_Process();           // 按键扫描
  AD_Process();            // ADC 滤波 / 过流停机
  Dcmotor_Process();       // 电机控制
  Lcd_Process();           // 液晶刷新
  PropertyReport_Process();// 开度属性上报（主循环执行，避免中断栈溢出）
  Gprs_Process();          // 4G 收发与状态机
  Eeprom_Wpara();          // 延迟写 EEPROM
  manage485();             // RS485 秒级查询
}
```

| 任务 | 主要文件 | 作用 |
|------|----------|------|
| `KEY_Process` | `HARDWARE/KEY` | 消抖后得到 `KeyNum` |
| `AD_Process` | `HARDWARE/ADC` | 电机电流等 → `armReg`；过流则停机 |
| `Dcmotor_Process` | `HARDWARE/DC_MOTOR` | 键控 + 目标开度跟踪 + 行程校准 |
| `Lcd_Process` | `HARDWARE/Lcd12864` | 背光 + 事件驱动刷屏 |
| `PropertyReport_Process` | `HARDWARE/GPRS/property_report.c` | 开度变化/停机强制属性上报 |
| `Gprs_Process` | `HARDWARE/GPRS` | 解析下行 JSON、状态机、周期推送、TX 队列 |
| `Eeprom_Wpara` | `HARDWARE/REG` | `Witeflag` 触发后按地址写入 |
| `manage485` | `HARDWARE/485` | 定时发读命令并收包 |
| `sbf20_poll` | `HARDWARE/USART/sbf20*.c` | 近场帧收发与应用层 |

## 4. 时基与传感（TIM2）

`HARDWARE/TIMER/timer.c` 中 TIM2 约 **5ms** 一次：

| 周期 | 动作 |
|------|------|
| 每 5ms | `JudeCount()` 编码器边沿计数；`JudeDir()` 限位判断 |
| 约 10ms | `JudeLocked()` 等辅助判断；置位 `flag10ms` 等 |
| 约 500ms | `JudeKaidu()` 电机停稳后固化开度并标记写 EEPROM |
| 持续 | `sys_ms += 5`，供通信节流、周期上报等使用 |

开度链路（详见 `qibiji/闸门开度统计流程说明.md`）：

```
光电编码器 CODE1/CODE2
        ↓ 边沿
   JudeCount → ChangeReality
        ↓
  Num_count（圈数）→ range_reality / gate_kaidu = Num_count * 100 / gate_maxturns
        ↓ 停稳
   JudeKaidu → gate_kaidu_count → EEPROM
```

限位：

- **上限位** → `FLAG_OPENALL = 1`（校准模式下可更新 `gate_maxturns`）
- **下限位** → `FLAG_CLOSEALL = 1`，`Num_count = 0`

## 5. 电机控制逻辑

文件：`HARDWARE/DC_MOTOR/dc_motor.c`。

### 5.1 状态与输出

| 常量（`main.h`） | 含义 |
|------------------|------|
| `MOTOR_UP` (0x01) | 上升 |
| `MOTOR_DOWN` (0x02) | 下降 |
| `MOTOR_STOP` (0x03) | 停止 |
| `MOTOR_RESET` (0x04) | 复位相关 |

`sys_moter_state` 驱动 `dcmotor_up` / `dcmotor_down` / `dcmotor_stop`（GPIOD 继电器时序）。

`Dcmotor_Process()` = `dcmotor_keycheck` → `dcmotor_ctrl` → `dcmotor_auto`。

### 5.2 本地控制

- 仅在 **首页**（`PageSta == 0` / `ViewNum == 0`）时，UP / DOWN / ENTER 才能改变电机状态
- 键值由 `KEY_Process` 提供，在 `dcmotor_keycheck` 中消费

### 5.3 远程控制

- 云端 JSON `setting`（`gprs_JSON.c`）写入 `armReg.range_target`，置 `remoteflag`
- `dcmotor_ctrl` 按「当前开度 vs 目标开度」决定升/降
- 到达目标（上升：`range_reality >= range_target`；下降：`<=`）则停机，并可触发属性上报 / 事件

### 5.4 自动停机条件（摘要）

- 下限位全关、上限位全开
- 远程到达目标开度
- ADC 过流（在 `AD_Process` 侧停机）

### 5.5 行程自动校准

`autoFlag` 有效时 `dcmotor_auto` 走状态机：下降到全关 → 上升到全开（记录 `gate_maxturns`）→ 再下降到全关。校准开关可在 LCD 设置页操作。

## 6. 人机界面（LCD + 键）

文件：`HARDWARE/Lcd12864/Dispaly.c` 等。左右键翻页，`ViewNum` 约 0～4：

| ViewNum | 页面 | 内容要点 |
|---------|------|----------|
| 0 | `homepage` | 开度、电机状态、电流/电压；本地升降有效 |
| 1 | `setpage` | 最大高度、堵转电流、校准开关等 |
| 2 | `G4gpage` | 在线状态、信号、CCID |
| 3 | `Devicepage` | 设备/版本信息 |
| 4 | `OperatePage` | 操作相关页 |

刷新策略为**事件驱动**（数据变化或翻页才刷），见 `LCD事件驱动刷新方案.md`。

---

## 7. 云端通信（4G / MQTT / JSON）

| 层次 | 文件 | 说明 |
|------|------|------|
| 硬件脚 | `HARDWARE/4G` | 复位、链路指示；`gprsonlineflag` |
| AT 状态机 | `gprs_AT.c` | 离线 → 初始化 → NORMAL |
| JSON 业务 | `gprs_JSON.c` | 解析 `enquiry` / `setting` / `version` / `update` 等 |
| 属性上报 | `property_report.c` | 开度阈值（如 2%）+ 最小间隔；停机 Force |
| 事件上报 | `event_pub.c` | 启停、限位、到达目标、报警等 JSON 入队 |
| 收发队列 | `uart_queue.c` | AT/JSON 统一串口队列，避免阻塞 |

典型下行：服务器 `setting` → 改目标开度 → 电机跟踪。  
典型上行：周期 clock、属性 `gateproperty`、事件 `gateevent`。

中断路径只置位 `PropertyReport_SetPending()`，真正组 JSON 在主循环 `PropertyReport_Process()`，避免栈溢出。

## 8. 近场与 RS485

### 8.1 近场 SBF-20（UART3）

- 传输层：固定约 20 字节二进制帧（`sbf20.c`）
- 应用层：JSON `op: read|write`（`sbf20_app.c`），可写采样间隔、上报周期等并落 EEPROM  
- 协议说明：`近场20B二进制帧串口协议-v2.0-传输层/应用层`

### 8.2 RS485（UART4）

`manage485` 按秒发送读寄存器命令；接收侧 CRC 校验后，电池/太阳能等字段解析当前多为注释预留，与 `armReg` 中 bat/solar 字段对应。

---

## 9. 寄存器与 EEPROM

结构体 `Register`（`reg.h`）是运行参数镜像，核心字段包括：

| 字段 | 含义 |
|------|------|
| `motor_on_off` | 电机命令 |
| `range_target` | 远程目标开度 |
| `motor_curr` 等 | 模拟量 |
| `gate_maxturns` | 全开对应圈数（满量程） |
| `gate_kaidu` / `gate_kaidu_count` | 固化开度与对应计数 |
| `motor_warning_curr_upper` | 过流报警阈值 |
| `gate_maxheight` | 全开高度(mm) |
| `reportT` | 周期上报间隔（秒，>0 时覆盖默认） |

启动 `para_read` 从 AT24CXX 载入；运行中 `Witeflag` + `EepromAddr` 由 `Eeprom_Wpara` 延迟写入，减少频繁擦写。

## 10. 端到端数据流

```
编码器 / 限位 ──TIM2──► Num_count、FLAG_*、开度
按键 / 云端 setting ──► sys_moter_state / range_target
                              │
                              ▼
                     dcmotor_up/down/stop → 继电器 → 电机
                              │
         ┌────────────────────┼────────────────────┐
         ▼                    ▼                    ▼
       LCD 显示         属性/事件上报           EEPROM 固化
                              │
                              ▼
                         UART2 → 4G MQTT
```

旁路：UART3 近场改配置；UART4 RS485 读外部电源类数据（预留）。

## 13. 一句话总结

启闭机固件以 **TIM2 传感时基** 维护开度与限位，以 **电机状态机** 统一本地键与远程目标，以 **armReg + EEPROM** 保存配置与固化开度，以 **4G JSON 队列** 完成云端控制与属性/事件上报，并以 **近场 SBF-20** 做现场参数配置。主循环只做非阻塞任务调度，重逻辑避免放在中断里执行。
