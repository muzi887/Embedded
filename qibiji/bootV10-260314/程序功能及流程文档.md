# STM32 BootLoader程序功能及流程文档

## 1. 项目概述

本项目是一个基于STM32F10x系列微控制器的BootLoader（引导加载程序）系统，主要功能是实现OTA（Over-The-Air）固件更新和应用程序跳转。系统采用双区备份机制，确保固件更新的安全性和可靠性。

### 1.1 硬件平台
- **MCU型号**: STM32F103VET6 / STM32F103VCT6
- **Flash容量**: 512KB / 256KB
- **RAM容量**: 64KB / 48KB
- **Flash页大小**: 2KB

### 1.2 主要功能模块
- BootLoader引导程序
- OTA固件更新
- 4G通信模块
- EEPROM存储（24C02）
- 看门狗保护
- 直流电机控制接口

## 2. Flash存储分区规划

### 2.1 分区布局（以512KB Flash为例）

| 分区名称 | 起始地址 | 页数 | 大小 | 用途 |
|---------|---------|------|------|------|
| Boot区(B) | 0x08000000 | 20页 | 40KB | BootLoader程序区 |
| A1区 | 0x0800A000 | 100页 | 200KB | 应用程序主区 |
| A2区 | 0x0803C000 | 100页 | 200KB | 应用程序备份区 |

### 2.2 地址定义

```c
#define ST32_FLASH_SADDR    0x08000000    // Flash起始地址
#define ST32_PAGE_SIZE      2048          // Flash页大小（2KB）
#define ST32_B_PAGE_NUM     20            // Boot区页数（40KB）
#define ST32_A1_SADDR       0x0800A000    // A1区起始地址
#define ST32_A2_SADDR       0x0803C000    // A2区起始地址
```

### 2.3 OTA标志定义

```c
#define OTA_A1_FLAG        0xA1A1A1A1    // 运行A1区标志
#define OTA_A2_FLAG        0xA2A2A2A2    // 运行A2区标志
```

## 3. 核心数据结构

### 3.1 OTA信息结构体

```c
typedef struct {
    u32 OTA_flag;           // OTA标志位，指示当前运行哪个区
    u8  OTA_ver[32];        // 固件版本号
    u32 OTA_size;           // OTA固件大小
} OTA_InfoCB;
```

**存储位置**: EEPROM地址200（OTA_ADDR）

### 3.2 更新缓冲区结构体

```c
typedef struct {
    u8  Updatabuff[ST32_PAGE_SIZE];  // 更新缓冲区（2KB）
    u8  BlockNB;                      // 块数量
    u16 size;                         // 最后一块的字节数
} UpDataA_CB;
```

## 4. 程序流程分析

### 4.1 主程序流程（main.c）

```
开始
  ↓
初始化延时系统
  ↓
配置NVIC中断优先级
  ↓
初始化UART1（460800波特率，PC通信）
  ↓
初始化UART2（115200波特率，4G模块）
  ↓
初始化LED端口
  ↓
初始化直流电机IO
  ↓
初始化IIC接口（用于EEPROM）
  ↓
从EEPROM读取OTA信息
  ↓
执行BootLoader分支判断
  ↓
初始化4G模块
  ↓
初始化看门狗（4秒，1990重载值）
  ↓
进入主循环（喂狗）
```

### 4.2 BootLoader分支流程（BootLoader_Brance）

```
开始
  ↓
读取OTA_Info.OTA_flag
  ↓
判断标志位
  ├─→ OTA_A1_FLAG?
  │     ↓ 是
  │   跳转到A1区（LOAD_A）
  │     ↓
  │   结束
  │
  └─→ 否
        ↓
      计算块数量（Cal_Block）
        ↓
      更新A1区（Update_A1）
        ↓
      设置OTA_flag = OTA_A1_FLAG
        ↓
      写入EEPROM
        ↓
      跳转到A1区（LOAD_A）
        ↓
      结束
```

### 4.3 应用程序跳转流程（LOAD_A）

```
开始
  ↓
读取目标地址的SP值（栈指针）
  ↓
验证SP值是否在RAM范围内
  ├─→ 0x20000000 ~ 0x2000FFFF?
  │     ↓ 是
  │   保存当前VTOR值
  │     ↓
  │   设置MSP = 目标地址的SP值
  │     ↓
  │   获取复位向量地址（addr+4）
  │     ↓
  │   清除BootLoader使用的资源
  │     ↓
  │   跳转到应用程序复位向量
  │     ↓
  │   结束
  │
  └─→ 否
        ↓
       打印错误信息
        ↓
       结束
```

### 4.4 OTA更新流程（Update_A1）

```
开始
  ↓
计算需要更新的块数（Cal_Block）
  ↓
循环处理每个块（除最后一块）
  ├─→ 从A2区读取一页数据（2KB）
  │     ↓
  │    写入A1区对应位置
  │     ↓
  └─→ 继续下一块
  ↓
处理最后一块
  ├─→ 从A2区读取剩余数据
  │     ↓
  │    写入A1区对应位置
  │     ↓
  └─→ 完成
  ↓
结束
```

### 4.5 块数量计算流程（Cal_Block）

```
开始
  ↓
判断OTA_size是否能被页大小整除
  ├─→ 能整除？
  │     ↓ 是
  │    BlockNB = OTA_size / PAGE_SIZE
  │    size = PAGE_SIZE / 2
  │     ↓
  │    结束
  │
  └─→ 否
        ↓
       BlockNB = OTA_size / PAGE_SIZE + 1
       size = OTA_size % PAGE_SIZE
        ↓
       判断size是否为偶数
        ├─→ 是偶数？
        │     ↓ 是
        │    size = size / 2
        │     ↓
        │    结束
        │
        └─→ 否
              ↓
             size = size / 2 + 1
              ↓
             结束
```

## 5. 关键函数说明

### 5.1 BootLoader_Brance()
**功能**: BootLoader分支判断，决定是直接跳转还是先更新再跳转
**参数**: 无
**返回值**: 无
**说明**: 根据EEPROM中的OTA_flag决定执行路径

### 5.2 LOAD_A(uint32_t addr)
**功能**: 跳转到指定的应用程序区
**参数**: 
- `addr`: 应用程序起始地址
**返回值**: 无
**说明**: 
- 验证栈指针有效性
- 设置MSP寄存器
- 跳转到应用程序复位向量

### 5.3 MSR_MSP(uint32_t addr)
**功能**: 汇编函数，设置主栈指针（MSP）
**参数**: 
- `addr`: 栈指针地址
**返回值**: 无
**说明**: 使用ARM汇编指令直接操作MSP寄存器

### 5.4 Update_A1()
**功能**: 将A2区的固件更新到A1区
**参数**: 无
**返回值**: 无
**说明**: 
- 从A2区读取数据
- 写入A1区对应位置
- 按页（2KB）为单位进行更新

### 5.5 Cal_Block()
**功能**: 计算OTA更新需要的块数量和最后一块的大小
**参数**: 无
**返回值**: 无
**说明**: 根据OTA_size计算更新参数，结果存储在UpDataA结构体中

### 5.6 BootLoader_Clear()
**功能**: 清除BootLoader使用的资源
**参数**: 无
**返回值**: 无
**说明**: 
- 反初始化USART1和USART2
- 为应用程序释放资源

### 5.7 AT24CXX_ReadOTAInfo()
**功能**: 从EEPROM读取OTA信息
**参数**: 无
**返回值**: 无
**说明**: 
- 检查EEPROM是否存在
- 如果不存在，设置默认值（OTA_A1_FLAG）
- 如果存在，读取OTA信息到OTA_Info结构体

## 6. 通信接口

### 6.1 UART1
- **波特率**: 460800
- **用途**: PC调试通信
- **功能**: 打印调试信息，监控程序运行状态

### 6.2 UART2
- **波特率**: 115200
- **用途**: 4G模块通信
- **功能**: 接收OTA更新数据，远程控制

### 6.3 IIC接口
- **用途**: EEPROM（24C02）通信
- **功能**: 存储和读取OTA配置信息

## 7. 安全机制

### 7.1 看门狗保护
- **类型**: 独立看门狗（IWDG）
- **超时时间**: 约4秒（4分频，1990重载值）
- **功能**: 防止程序死机，自动复位系统

### 7.2 双区备份机制
- **A1区**: 当前运行的应用程序
- **A2区**: 备份的应用程序
- **优势**: 更新失败时可回退到备份区

### 7.3 栈指针验证
- 跳转前验证目标地址的栈指针
- 确保在有效RAM范围内（0x20000000 ~ 0x2000FFFF）
- 防止跳转到无效地址导致系统崩溃

## 8. 工作模式

### 8.1 正常启动模式
1. 系统上电
2. BootLoader初始化硬件
3. 读取EEPROM中的OTA标志
4. 如果标志为OTA_A1_FLAG，直接跳转到A1区
5. 应用程序开始运行

### 8.2 OTA更新模式
1. 系统上电
2. BootLoader初始化硬件
3. 读取EEPROM中的OTA标志
4. 如果标志不是OTA_A1_FLAG（或需要更新）
5. 从A2区读取固件数据
6. 写入A1区
7. 更新OTA标志为OTA_A1_FLAG
8. 写入EEPROM
9. 跳转到A1区运行

## 10. 注意事项

### 10.1 Flash操作注意事项
- Flash写入前必须先擦除
- 按页（2KB）为单位进行操作
- 写入时以半字（16位）为单位
- 操作期间需要关闭中断

### 10.2 跳转注意事项
- 跳转前必须清除BootLoader使用的资源
- 确保目标应用程序的栈指针有效
- 跳转后BootLoader代码不再执行

### 10.3 EEPROM操作注意事项
- OTA信息存储在地址200（OTA_ADDR）
- 写入前需要检查EEPROM是否存在
- 结构体大小为固定值（OTA_INFOCB_SIZE）

### 10.4 看门狗注意事项
- 必须在4秒内喂狗
- 主循环中需要定期调用IWDG_Feed()
- 看门狗一旦启动无法停止

## 14. 总结

本BootLoader系统实现了以下核心功能：

1. **安全启动**: 通过验证栈指针确保跳转安全
2. **OTA更新**: 支持双区备份的固件更新机制
3. **状态保存**: 使用EEPROM保存OTA配置信息
4. **看门狗保护**: 防止程序死机
5. **通信支持**: 支持PC调试和4G远程通信
6. **资源管理**: 跳转前清理资源，避免冲突

系统设计合理，具有良好的可靠性和可维护性，适用于需要远程更新的嵌入式应用场景。
